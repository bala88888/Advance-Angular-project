Angular Summary

1. What is Angular?

Angular is a platform and framework for building single-page client applications using HTML and TypeScript. It’s developed by Google and offers a component-based architecture.

2. Components

Components are the building blocks of Angular applications. Each component consists of:

  • Template: The HTML view.
  • Class: The TypeScript code defining behavior.
  • Styles: CSS for that component.

3. Modules

Angular apps are modular. The root module is AppModule, which declares components and imports other modules.

4. Data Binding

Angular provides various ways to bind data:

  • Interpolation: Display data in template using {{ value }}
  • Property Binding: Bind element properties with [property]="value"
  • Event Binding: Listen to events with (event)="handler()"
  • Two-way Binding: Using [(ngModel)]="value" for syncing data

5. Directives

Directives add behavior to elements:

  • Structural: Change DOM layout, e.g., *ngIf, *ngFor
  • Attribute: Change appearance or behavior, e.g., [ngClass], [ngStyle]

6. Services and Dependency Injection

Services are used to share data and logic across components. Angular uses dependency injection to provide these services.

7. Routing

Angular Router enables navigation between views/components. You configure routes in app-routing.module.ts and use <router-outlet>.

8. Pipes

Pipes transform output in templates, e.g., {{ dateValue | date }} formats dates.

9. Lifecycle Hooks

Components have lifecycle hooks like ngOnInit() for initialization and ngOnDestroy() for cleanup.

10. Forms

Angular supports template-driven and reactive forms for handling user input and validation.

Summary

Angular is a powerful framework for building SPAs with modular components, data binding, routing, and much more. This summary covers core concepts you’ll use daily.
